		                    <p><p>&quot;size does not show up if we say sizeof(object) of the class?&quot;</p>
<p>It is not part of the allocated memory when an object is created, otherwise it would not make sense, as that would mean a new static member would have space for each object creation. &nbsp;This all stays consistent for memory allocations using malloc(sizeof(Class)) etc, as sizeof should not return the size of the class, but the size of an object to created. &nbsp;(As far as I know anyhow).</p>
<p>I think static members are simply a global variable that is resolved in the name space of the class (e.g. Class::static_var)</p>

		                <div class="commentsbody">
		                    <p><p>&quot;size does not show up if we say sizeof(object) of the class?&quot;</p>
<p>It is not part of the allocated memory when an object is created, otherwise it would not make sense, as that would mean a new static member would have space for each object creation. &nbsp;This all stays consistent for memory allocations using malloc(sizeof(Class)) etc, as sizeof should not return the size of the class, but the size of an object to created. &nbsp;(As far as I know anyhow).</p>
<p>I think static members are simply a global variable that is resolved in the name space of the class (e.g. Class::static_var)</p>
